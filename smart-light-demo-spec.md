## 1. システム概要

人感センサーの検知状態に応じて照明を自動ON/OFFし、照度センサーで輝度を自動調整することで省エネを実現するIoT照明シミュレーターのデモ版。

デモ版のため実IoTデバイスとは接続しない。センサー値はUIのスライダー・ボタンで手動シミュレートする。

### アーキテクチャ

```
[ブラウザ UI]
  Next.js（TypeScript）
       |
       | HTTP / REST
       v
[バックエンドAPI]
  Python FastAPI
       |
       v
  [SQLite DB]
  （毎日JST 03:00 自動リセット）
```

> **選定理由：ウェブ**
> IoT制御ダッシュボードはリアルタイム表示・グラフ可視化が主目的であり、複数デバイスのブラウザアクセスが想定されるためウェブを選択。
> デモ版のためFastAPI + SQLiteの構成とし、外部API・認証は一切不使用。

## 2. 制約・前提条件

### デモ版固有制約

| 制約項目 | 内容 |
|---------|------|
| 認証 | なし（セッションIDで代替） |
| 外部API | 一切使用禁止 |
| IoT通信 | なし（UIシミュレーション） |
| AI機能 | なし（ルールベースのみ） |
| DB | SQLiteのみ |
| DBリセット | JST毎日03:00に全データ削除 |
| セッション | Cookie + SQLiteで管理。セッションIDをオーナーキーとして全テーブルに付与 |
| 個人情報 | 氏名・メール・生年月日・住所・電話番号は一切保存しない |
| Bot対策 | ハニーポット方式（reCAPTCHA禁止） |
| 実装形式 | 1 issueのワンショット実装（`claude --dangerously-skip-permissions`） |

### テスト制約

> ⚠️ **デモ版は最小単位のデータでしかテストできない**
>
> デモ版はSQLiteを使用し、シングルユーザー・シングルセッションの最小構成で動作する。
> 以下のテストはデモ版では実施不可能であり、MVP以降で対応する：
> - マルチユーザー同時接続のテスト
> - 実IoTデバイス（MQTT/HTTP）との結合テスト
> - 大量センサーデータの性能テスト（ロードテスト）
> - 長期間データ蓄積後の省エネ統計精度テスト

## 3. マスタデータ

デモ版におけるマスタデータの件数を以下に明示する。

| マスタ名 | 件数 | 内容 |
|---------|------|------|
| 制御モード | 2件 | `auto`（センサー自動）/ `manual`（手動） |
| 照明状態 | 2件 | `on` / `off` |
| センサー状態 | 2件 | `detected`（検知あり）/ `none`（検知なし） |
| 消灯時間帯プリセット | 3件 | 深夜帯（00:00-06:00）/ 就寝帯（22:00-07:00）/ カスタム |
| デフォルト設定 | 1件 | 待機時間60秒、目標照度300lux、輝度70% |

**合計マスタデータ：10件**

> すべてアプリケーションコード内にハードコードされ、DBには保存しない。
> ユーザーが変更した設定値のみDBに保存する（セッションID紐付き）。

## 4. 機能仕様

### F1: センサーシミュレーション

- UIボタン「検知あり／なし」でセンサー状態をトグル
- 複数センサー（最大3箇所）を独立してON/OFF可能
- いずれか1つでも検知ありなら「在室」判定

### F2: デバウンス処理

- センサー状態変化がN秒（デフォルト2秒）継続した場合のみ確定
- 設定画面でデバウンス時間を1〜10秒で変更可能

### F3: 照明制御

- 在室確定 → 照明ON（設定輝度）
- 離席確定 → 待機タイマー開始 → タイマー満了で照明OFF
- 待機中に再検知 → タイマーリセット・照明ON継続

### F4: 輝度自動調整

- 照度センサー値（UIスライダー 0〜1000 lux）を入力
- 目標照度との差分で輝度を比例制御（P制御）
- 輝度範囲：0〜100%（0%は消灯扱い）

### F5: 手動制御・優先制御

- 手動モード時：センサー制御を停止し、UIの輝度スライダーで直接制御
- 手動モード解除 → 自動制御に復帰
- 手動OFF中はセンサー検知があっても点灯しない
- 手動ON中はセンサー無検知でも消灯しない

### F6: スケジュール制御

- 消灯時間帯を設定可能（HH:MM〜HH:MM）
- 消灯時間帯中はセンサー検知無効（照明OFF維持）
- 消灯時間帯終了後は自動制御に復帰

### F7: 省エネ計算・可視化

- 照明ON時間を1秒単位で累積記録
- 消費電力量（Wh）= ON時間（h）× 設定消費電力（W）
- 節約電力量 = 常時ON想定消費量 − 実消費量
- 折れ線グラフで時系列表示（過去24時間）

### F8: セッション管理

- 初回アクセス時にUUID v4のセッションIDを生成しCookieに保存（HttpOnly, SameSite=Strict）
- 全DBテーブルにsession_idカラムを持ち、クエリ時に必ずWHERE session_id = :sidを付与
- セッションIDが異なるデータへのアクセスは404を返す
